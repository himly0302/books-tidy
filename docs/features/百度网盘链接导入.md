## 开发方案

### 要实现什么

- 读取与 `books.json` 同级的 CSV 文件（文件名为书籍类型，如 `传记.csv`）
- 解析 CSV 中的文件名和链接，按 type+name 匹配数据库中的书籍
- 为匹配的书籍新增 `bd_link` 字段，写入对应的百度网盘链接
- 将更新后的数据库副本保存到 `result/` 目录，文件名格式为 `YYYYMMDD-{条目数量}.json`

### CSV 输入格式

```csv
文件名,链接,提取码,分享时间,分享状态
小文艺口袋文库·知人系列,https://pan.baidu.com/s/xxx?pwd=6yk7,6yk7,2026-04-30 18:14,生成成功
```

CSV 文件名即书籍类型，如 `传记.csv` → type 为 `传记`。

### 匹配逻辑

1. CSV 文件名（去扩展名）= 书籍 type
2. CSV 中"文件名"列 ≈ BookInfo.name（需模糊匹配，因为 CSV 中文件名可能包含冗余信息）

### 与现有功能的关系

- 依赖现有模块：database.ts - loadDatabase / saveDatabase 读写 JSON 数据库
- 集成位置：index.ts - 新增 `import-links` CLI 子命令
- 类型扩展：types.ts - BookInfo 接口新增 `bd_link?` 字段

### 新增依赖

无。CSV 格式简单，手动解析即可（按逗号分割，首行为表头）。

### 需要修改的文件

```
src/types.ts       # BookInfo 接口新增 bd_link? 字段
src/index.ts       # 新增 import-links 命令注册
.gitignore         # 新增 result/ 目录忽略
```

### 需要新增的文件



**步骤 2：核心实现**

- [ ] 创建 `src/import-links.ts`，实现以下功能：

```
import-links --dir <directory>
```

处理流程：
1. 扫描 `--dir` 目录下所有 `.csv` 文件，文件名（去扩展名）作为 type
2. 解析每个 CSV：跳过首行表头，提取"文件名"和"链接"两列
3. 加载 `--dir/books.json` 数据库
4. 遍历数据库书籍，按 type 找到对应 CSV 数据，按 name 匹配行
5. 匹配规则：CSV 文件名包含 BookInfo.name 即视为匹配（CSV 文件名可能是"书名（全X册）"格式，书名是子串）
6. 更新匹配书籍的 `bd_link` 字段
7. 保存更新后的 books.json
8. 创建 `result/` 目录，将更新后的完整 JSON 副本保存为 `YYYYMMDD-{books.length}.json`

- [ ] CSV 解析函数：按行分割，逗号分隔，首行跳过，返回 `{ name, link }[]`
- [ ] 匹配函数：遍历书籍，按 type+name 子串匹配
- [ ] 输出匹配统计：匹配成功数、未匹配数

**步骤 3：集成到 CLI**

- [ ] 修改 `src/index.ts`：新增 `import-links` 命令注册

```typescript
program.command('import-links')
  .description('从 CSV 文件导入百度网盘分享链接到数据库')
  .requiredOption('--dir <directory>', '包含 books.json 和 CSV 文件的目录')
```

## 约束与风险

- **匹配精度**：CSV 文件名可能包含冗余信息（如"书名（全X册）"），使用子串匹配可能误匹配。需在输出中展示匹配结果供人工确认
- **兼容性**：`bd_link` 为可选字段，不影响现有流程
- **性能/安全**：无风险，纯本地文件操作，无外部 API 调用
