## 开发方案

### 要实现什么

- 核心功能 1：列出账号下所有存储空间（`qiniu buckets`）
- 核心功能 2：列出指定空间内所有文件（`qiniu files --bucket <name>`）
- 核心功能 3：删除指定空间（先清空全部文件再删除空间）（`qiniu delete-bucket --bucket <name>`）
- 核心功能 4：提取七牛云共享客户端，供上传器和空间管理复用

### 与现有功能的关系

- 复用认证模式：`src/uploaders/qiniu.ts` 中 `Mac` + `Config` 的创建方式
- 复用环境变量：`QINIU_ACCESS_KEY`、`QINIU_SECRET_KEY`（已有）
- 集成位置：`src/index.ts`（注册 `qiniu` 命令组）
- 此功能与书籍整理逻辑无关，是独立的七牛云管理工具

### 新增依赖

无。已有 `qiniu`@^7.x SDK，其 `BucketManager` 提供 `listBucket`、`listPrefix`、`batch`、`deleteBucket` 等所需全部 API。

### 需要修改的文件

```
src/index.ts              # 注册 qiniu 命令组（buckets / files / delete-bucket）
src/uploaders/qiniu.ts    # 认证逻辑改为引用共享客户端（可选，降低耦合）
```

### 需要新增的文件

```
src/qiniu/client.ts          # 七牛云共享客户端（认证 + 配置）
src/qiniu/bucket-manager.ts  # 空间管理操作（列空间、列文件、批量删文件、删空间）
src/qiniu-manage.ts          # qiniu 命令编排器（注册子命令 + 交互流程）
```

## 实施步骤

**步骤 1：共享客户端（src/qiniu/client.ts）**

- [ ] 从环境变量读取 `QINIU_ACCESS_KEY`、`QINIU_SECRET_KEY`、`QINIU_ZONE`
- [ ] 导出 `getMac()` 返回 `qiniu.auth.digest.Mac` 实例
- [ ] 导出 `getConfig()` 返回 `qiniu.conf.Config` 实例
- [ ] 导出 `getBucketManager()` 返回 `qiniu.rs.BucketManager` 实例
- [ ] 参数校验：缺少 AK/SK 时 `throw new Error` 并给出中文提示

```typescript
// 设计参考
export function getMac(): qiniu.auth.digest.Mac
export function getConfig(): qiniu.conf.Config
export function getBucketManager(): qiniu.rs.BucketManager
```

**步骤 2：空间管理核心（src/qiniu/bucket-manager.ts）**

- [ ] 实现 `listBuckets()`：调用 `BucketManager.listBucket()` 列出所有空间名
- [ ] 实现 `listFiles(bucket, prefix?)`：分页遍历 `BucketManager.listPrefix()`，收集所有文件 key
  - 每页 limit=1000，根据 `nextMarker` 继续翻页
  - 支持 `prefix` 参数过滤前缀
- [ ] 实现 `batchDeleteFiles(bucket, keys)`：分批调用 `BucketManager.batch()` 删除文件
  - 每批最多 1000 个 key
  - 打印每批进度
- [ ] 实现 `deleteBucket(bucket)`：调用 `BucketManager.deleteBucket()` 删除空空间
- [ ] 所有方法将 callback 包装为 Promise，与现有 async/await 风格一致

**步骤 3：命令编排器（src/qiniu-manage.ts）**

- [ ] 实现 `listBucketsCommand()`：列出所有空间名及文件数量提示
- [ ] 实现 `listFilesCommand(options)`：列出指定空间的全部文件 key
  - 参数：`--bucket <name>`（必选）、`--prefix <prefix>`（可选）
- [ ] 实现 `deleteBucketCommand(options)`：删除指定空间
  - 参数：`--bucket <name>`（必选）
  - 交互流程：
    1. 列出空间内文件数量
    2. 确认提示：`确认删除空间 "xxx" 及其 N 个文件？(y/N)`
    3. 批量删除文件（显示进度）
    4. 删除空间本身
    5. 输出结果汇总

**步骤 4：注册命令（src/index.ts）**

- [ ] 导入 `qiniu-manage.ts` 中的命令函数
- [ ] 注册 `qiniu` 命令组及其 3 个子命令：

```typescript
const qiniuCmd = program.command('qiniu').description('七牛云空间管理');

qiniuCmd.command('buckets')
  .description('列出所有存储空间')
  .action(listBucketsCommand);

qiniuCmd.command('files')
  .description('列出空间内文件')
  .requiredOption('--bucket <name>', '空间名称')
  .option('--prefix <prefix>', '文件前缀过滤')
  .action(listFilesCommand);

qiniuCmd.command('delete-bucket')
  .description('删除存储空间（先清空文件再删除空间）')
  .requiredOption('--bucket <name>', '空间名称')
  .action(deleteBucketCommand);
```

## 约束与风险

- **新依赖**：无需新增依赖，完全复用已有 `qiniu` SDK
- **兼容性**：纯新增功能，不影响现有书籍整理流程；`src/qiniu/client.ts` 为共享模块，后续可让 `QiniuUploader` 也引用它以消除重复
- **安全风险**：删除操作不可逆，必须有确认提示；批量删除大空间时注意 API 频率限制（七牛批量操作限制每次最多 1000 个）
- **性能**：大空间（>10 万文件）的文件列表和删除耗时较长，需要分页+分批处理并显示进度
