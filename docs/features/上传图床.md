## 项目概况

**技术栈**：TypeScript 5.x + Commander 11 + 七牛云 Node.js SDK
**架构模式**：流水线式模块化（扫描 -> AI 分析 -> 整理 -> 存储）
**代码风格**：camelCase 命名 / 无注释 / 同步文件操作 + 异步 AI 调用

## 改动点

### 要实现什么
- 核心功能 1：定义通用图床上传接口（`ImageUploader`），支持多图床提供商接入
- 核心功能 2：新增 `upload-pics` CLI 子命令，读取 `books.json`，批量上传图片并回写 `picUrl`
- 核心功能 3：上传前自动压缩图片（保持质量，减小体积）
- 核心功能 4：跳过已有 `picUrl` 的书籍，支持断点续传
- 首个实现：七牛云对象存储（Kodo）

### 与现有功能的关系
- 依赖现有模块：`database.ts`（加载/保存 books.json）、`types.ts`（类型定义）
- 集成位置：`src/index.ts`（注册新命令）
- 数据流：读取 `output/books.json` -> 逐本处理 pic 字段 -> 压缩 -> 上传到七牛云 -> 回写 picUrl

### 新增依赖
- `qiniu`@^7.x - 七牛云官方 Node.js SDK（提供上传凭证生成、文件上传、空间管理）
- `sharp`@^0.33 - 图片压缩处理（Node.js 最主流的图片处理库，支持 JPEG/PNG/WebP 压缩）

### 七牛云上传设计

```typescript
// src/uploaders/qiniu.ts

// 从环境变量读取配置
// QINIU_ACCESS_KEY - 七牛云 Access Key
// QINIU_SECRET_KEY - 七牛云 Secret Key
// QINIU_BUCKET     - 存储空间名称
// QINIU_DOMAIN     - CDN 加速域名（如 https://img.example.com）
// QINIU_ZONE       - 存储区域（默认 z0 华东）

// 上传流程：
// 1. 使用 Mac(accessKey, secretKey) 创建认证对象
// 2. 使用 PutPolicy 生成 uploadToken
// 3. 使用 FormUploader.putFile() 上传本地文件
// 4. 拼接 CDN 域名 + key 返回完整 URL

// remoteKey 命名规则：books/{type}/{name}/{hash}.jpg
// 与本地 pic 字段结构保持一致，便于映射
```

### 压缩策略

使用 `sharp` 将图片压缩为 JPEG 格式（quality: 80），最大宽度 1200px：
- JPEG：quality 80，mozjpeg 优化
- PNG：转 JPEG（书籍封面无需透明通道）
- 输出到临时文件，上传后删除

### 上传流程

```
遍历 books.json
  → 跳过 picUrl 非空的条目
  → 读取 pic 字段对应的本地文件路径（output + pic）
  → sharp 压缩图片到临时文件
  → 调用 QiniuUploader.upload(localPath, remoteKey) 上传到七牛云
  → 将返回的 URL 写入 picUrl 字段
  → 删除临时压缩文件
  → 每上传 5 本，自动保存一次 books.json（防止中途失败丢数据）
  → 全部完成后保存 books.json
```

## 注意事项

**技术风险**：
- `sharp` 依赖原生模块（libvips），Windows 下安装通常无问题（提供预编译二进制），但需注意 Node.js 版本兼容性
- 七牛云免费额度：10GB 存储 + 10GB/月 CDN 流量（每月），超出需付费
- 存储空间需绑定 CDN 加速域名才能通过 URL 访问文件

**兼容性**：`picUrl` 为可选字段，完全向后兼容，不影响现有 tidy / analyze 流程。

**扩展性**：新增图床只需在 `src/uploaders/` 下新增文件，实现 `ImageUploader` 接口即可。
