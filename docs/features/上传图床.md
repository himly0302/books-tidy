## 项目概况

**技术栈**：TypeScript 5.x + Commander 11 + 原生 fetch + sharp 图片压缩
**架构模式**：流水线式模块化（扫描 → AI 分析 → 整理 → 存储 → 上传图床）
**代码风格**：camelCase 命名 / 无注释 / 同步文件操作 + 异步 API 调用

---

## 改动点

### 要实现什么
- 核心功能 1：读取 `books.json` 数据库，将本地封面图片上传到 SM.MS 图床，获取公开 URL
- 核心功能 2：上传前对图片进行有损压缩（保持视觉质量），控制文件体积在 SM.MS 免费额度（5MB）内
- 核心功能 3：为每条书记录新增 `picUrl` 字段，存储图床 URL
- 核心功能 4：跳过已有 `picUrl`（非空）的记录，支持增量上传

### 与现有功能的关系
- 依赖现有模块：`database.ts`（读取/保存 books.json）、`types.ts`（BookInfo 类型）
- 新增独立模块：图片压缩 + SM.MS API 上传
- 集成位置：`src/index.ts`（注册 `upload-pics` 子命令）

### 新增依赖
- `sharp`@^0.33.x — 图片压缩（PNG/JPG/WebP → JPEG，控制质量与尺寸）

```
src/index.ts           # 注册 upload-pics 子命令

```

### 需要新增的文件
```
src/compressor.ts      # 图片压缩：读取本地图片，输出压缩后的 Buffer
src/uploader.ts        # SM.MS API 上传：压缩 → 上传 → 返回 URL
src/upload.ts          # upload-pics 命令编排：读数据库 → 过滤 → 逐本上传 → 更新数据库
```

### 实施步骤

**步骤 1：环境准备**
- [ ] 安装 `sharp` 依赖：`npm install sharp && npm install -D @types/sharp`
- [ ] 在 `.env.example` 中添加 `SMMS_API_TOKEN=your-token-here`
- [ ] 用户在 `.env` 中配置实际的 SMMS_API_TOKEN

**步骤 2：扩展类型定义（src/types.ts）**
- [ ] `BookInfo` 接口新增可选字段 `picUrl?: string`
- [ ] 新增 `UploadOptions` 接口：`{ input: string }`（input 为 books.json 所在目录）

**步骤 3：实现图片压缩（src/compressor.ts）**
- [ ] 导出 `compressImage(filePath: string): Buffer`
- [ ] 使用 sharp 将图片转为 JPEG 格式，quality 80，最大边 1200px
- [ ] 返回压缩后的 Buffer（不写临时文件，直接在内存中处理）

**步骤 4：实现图床上传（src/uploader.ts）**
- [ ] 导出 `uploadToSmms(imageBuffer: Buffer, filename: string): Promise<string>`
- [ ] 使用原生 `fetch` + `FormData` + `Blob` 发送 POST 到 `https://smms.app/api/v2/upload`
- [ ] Header：`Authorization: <SMMS_API_TOKEN>`
- [ ] 解析响应，提取 `data.url` 返回
- [ ] 错误处理：API 错误、网络超时、上传失败重试（最多 3 次，间隔 2 秒，复用 analyzer.ts 的重试模式）

**步骤 5：实现命令编排（src/upload.ts）**
- [ ] 导出 `uploadPics(options: UploadOptions): Promise<void>`
- [ ] 流程：
  1. 校验 `SMMS_API_TOKEN` 环境变量存在
  2. 校验 input 目录存在
  3. 调用 `loadDatabase(input)` 读取 books.json
  4. 过滤出有 `pic` 且 `picUrl` 为空的记录
  5. 对每条记录：
     - 拼接本地图片路径：`path.join(input, book.pic)`
     - 调用 `compressImage()` 压缩
     - 调用 `uploadToSmms()` 上传
     - 更新 `book.picUrl = url`
     - 打印进度：`[i/total] 《书名》上传成功: url`
  6. 调用 `saveDatabase()` 保存更新后的数据库
  7. 打印汇总：成功 N 本，跳过 N 本，失败 N 本

**步骤 6：注册 CLI 命令（src/index.ts）**
- [ ] 添加 `upload-pics` 子命令，选项 `-i, --input <dir>`（必填）
- [ ] 使用说明：`npm run dev -- upload-pics -i <output目录>`

## 注意事项

**技术风险**：
- `sharp` 是原生 addon，安装时需编译。Windows 上通常需安装 `node-gyp` 和 Visual Studio Build Tools。若安装失败，可使用 `@img/sharp-win32-x64` 替代
- SM.MS 免费版单文件限制 5MB，压缩后通常可满足要求
- SM.MS API 有请求频率限制，批量上传时需控制并发（串行上传即可）

**兼容性**：
- `BookInfo` 新增 `picUrl` 可选字段，向后兼容，已有的 `books.json` 无需迁移
- SM.MS 图床长期可用性取决于第三方服务，建议用户定期备份 `books.json`
