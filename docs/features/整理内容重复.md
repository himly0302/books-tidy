## 开发方案

### 要实现什么

- 核心功能 1：整理前过滤已处理书籍，避免重复调用 AI 和重复写入文件
- 核心功能 2：`BookInfo` 新增 `sourceFolder` 字段，记录原始文件夹名作为去重依据
- 核心功能 3：全局历史记录文件，跨 output 目录去重并统计所有整理操作

### 与现有功能的关系

- 依赖现有模块：`src/database.ts` — 已有 `loadDatabase`、`addBooks`、`hashName`
- 集成位置：`src/tidy.ts:18-27` — 在 AI 分析之前插入去重过滤逻辑
- 影响：`addBooks` 签名需增加 `inputDir` 参数，以计算确定性 ID 和记录来源

### 去重策略

采用两级去重，覆盖所有场景：

| 层级 | 去重依据 | 时机 | 覆盖场景 |
|------|---------|------|---------|
| 本地去重 | `books.json` 中的 `sourceFolder` | AI 分析前 | 同一 input/output 重复执行 |
| 全局去重 | `~/.books-tidy/history.json` 中的 `(inputDir, folderName)` | AI 分析前 | 不同 input/output 路径 |
| 名称去重 | 标准化后的 `name+author` | AI 分析后（`addBooks` 内） | AI 返回名称不一致（如带《》）、旧记录无 `sourceFolder` |

**标准化规则**：去除 `《》「」""''`、去除首尾空白、统一全角/半角符号。

### 新增依赖

无。完全使用 Node.js 内置 `crypto`、`fs`、`path` 模块。

```typescript
export function normalizeBookName(name: string): string {
  return name
    .replace(/[《》「」""''\s]/g, '')
    .replace(/（/g, '(')
    .replace(/）/g, ')')
    .trim();
}
```

- [ ] 新增 `filterDuplicateBooks(db, books, inputDir): { newBooks, skipped }` — 本地去重过滤

```typescript
export interface DedupResult {
  newBooks: BookRaw[];
  skipped: BookRaw[];  // 被过滤的已处理书籍
}

export function filterDuplicateBooks(
  db: BooksDatabase,
  books: BookRaw[],
  inputDir: string
): DedupResult {
  // 1. 构建已有 sourceFolder 集合（精确匹配）
  const existingFolders = new Set(
    db.books.filter(b => b.sourceFolder).map(b => b.sourceFolder)
  );

  // 2. 构建已有标准化名称集合（模糊匹配，兜底旧记录）
  const existingNames = new Set(
    db.books.map(b => normalizeBookName(b.name) + ':' + b.author)
  );

  const newBooks: BookRaw[] = [];
  const skipped: BookRaw[] = [];

  for (const book of books) {
    // 精确匹配：sourceFolder
    if (existingFolders.has(book.folderName)) {
      skipped.push(book);
      continue;
    }
    newBooks.push(book);
  }

  return { newBooks, skipped };
}
```

> 注意：名称级别的模糊匹配推迟到 `addBooks` 内执行，因为需要 AI 分析结果。此处仅做 `sourceFolder` 精确匹配，以避免不必要的 AI 调用。

- [ ] 修改 `addBooks` — 记录 `sourceFolder`，新增名称级去重

```typescript
export function addBooks(
  db: BooksDatabase,
  books: BookRaw[],
  analyses: AIAnalysisResult[],
): BooksDatabase {
  const newDb = { books: [...db.books] };

  // 构建已有标准化名称集合（兜底旧记录和 AI 名称不一致）
  const existingKeys = new Set(
    newDb.books.map(b => normalizeBookName(b.name) + ':' + b.author)
  );

  for (let i = 0; i < books.length; i++) {
    const book = books[i];
    const analysis = analyses[i];
    const nameKey = normalizeBookName(analysis.name) + ':' + analysis.author;

    // 名称级去重：跳过已存在的书籍
    if (existingKeys.has(nameKey)) continue;
    existingKeys.add(nameKey);

    const ext = book.pics[0] ? path.extname(book.pics[0]) : '.jpg';
    const picHash = hashName(analysis.name) + ext;

    const entry: BookInfo = {
      id: uuidv4(),
      name: analysis.name,
      author: analysis.author,
      type: analysis.type,
      pic: `${analysis.type}/${analysis.name}/${picHash}`,
      sourceFolder: book.folderName,
      addedAt: new Date().toISOString(),
    };
    newDb.books.push(entry);
  }

  return newDb;
}
```

```typescript
export async function tidyCommand(options: TidyOptions) {
  const inputDir = path.resolve(options.input);
  const outputDir = path.resolve(options.output);

  // ... 目录校验不变 ...

  console.log('Scanning books...');
  let books = scanBooks(inputDir);
  if (books.length === 0) {
    console.log('No book folders found.');
    return;
  }
  console.log(`Found ${books.length} book folders.`);

  // === 新增：去重过滤 ===
  // 1. 本地去重（基于 output 目录的 books.json）
  const dbPath = path.join(outputDir, 'books.json');
  const db = loadDatabase(dbPath);
  const localResult = filterDuplicateBooks(db, books, inputDir);
  if (localResult.skipped.length > 0) {
    console.log(`Skipped ${localResult.skipped.length} already processed books (local).`);
  }

  // 2. 全局历史去重
  const globalResult = filterByGlobalHistory(localResult.newBooks, inputDir);
  if (globalResult.skipped.length > 0) {
    console.log(`Skipped ${globalResult.skipped.length} already processed books (global history).`);
  }

  books = globalResult.newBooks;
  if (books.length === 0) {
    console.log('All books are already processed. Nothing to do.');
    return;
  }
  console.log(`Processing ${books.length} new books.`);
  // === 去重结束 ===

  console.log('Analyzing with AI...');
  const analyses = await analyzeBooks(books);
  // ... 后续流程不变（打印、整理、保存）...

  // addBooks 内部做名称级去重
  const newDb = addBooks(db, books, analyses);
  saveDatabase(dbPath, newDb);

  // 记录到全局历史
  for (let i = 0; i < books.length; i++) {
    recordProcessed(inputDir, outputDir, books[i].folderName, analyses[i].name);
  }

  console.log(`Done! ${books.length} books organized. Database saved to ${dbPath}`);
}
```

## 约束与风险

- **新依赖**：无需新增，完全使用 Node.js 内置模块
- **兼容性**：`sourceFolder` 为可选语义（旧记录没有），去重逻辑对旧记录使用名称匹配兜底；不破坏现有 `upload-pics`、`analyze` 等命令
- **性能**：去重过滤为内存 Set 查找（O(1)），对数万级别书籍无性能影响
- **全局历史文件**：`~/.books-tidy/history.json` 随使用增长，但单条记录极小（约 100 字节），即使处理 10 万本书也仅约 10MB
- **名称标准化局限性**：无法处理完全不同的书名指代同一本书的情况（如"三体"与"地球往事"），但这类情况极罕见
