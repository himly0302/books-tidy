## 开发方案

### 要实现什么

- 将新数据中的 `brief` 字段按 `sourceFolder` 匹配，迁移到旧数据中
- 旧数据保持为基准，仅追加 `brief` 字段
- 一次性处理，完成后输出合并统计

### 数据对比

| 字段 | 旧数据 (20260501-452.json) | 新数据 (books.json) |
|------|---------------------------|---------------------|
| brief | ❌ 缺失 | ✅ 有 |
| bd_link | ✅ 有 | ❌ 缺失 |
| picUrl | ✅ 有 | ❌ 缺失 |
| 匹配键 | sourceFolder（完全一致） | sourceFolder |

### 与现有功能的关系

- 纯数据迁移脚本，不影响现有命令和流水线
- 匹配逻辑复用 `sourceFolder` 字段（与 database.ts 去重逻辑一致）

## 实施步骤

**步骤 1：核心实现**

- [ ] 创建 `src/merge.ts`

```
merge --old <旧json> --new <新json>
```

处理流程：
1. 加载旧 JSON 和新 JSON
2. 按 `sourceFolder` 建立新数据索引
3. 遍历旧数据，查找匹配的新数据
4. 匹配成功则将 `brief` 写入旧数据
5. 保存合并后的旧数据（覆盖原文件）
6. 输出统计：匹配成功数、未匹配数

**步骤 2：集成到 CLI**

- [ ] 修改 `src/index.ts`：新增 `merge` 命令

```typescript
program.command('merge')
  .description('将新数据中的 brief 字段合并到旧数据')
  .requiredOption('--old <file>', '旧数据 JSON 文件（基准，保持不变）')
  .requiredOption('--new <file>', '新数据 JSON 文件（来源 brief）')
  .action(mergeCommand);
```

## 约束与风险

- **数据安全**：旧数据仅追加 brief 字段，不修改其他字段；建议合并前备份
- **匹配可靠性**：sourceFolder 在新旧数据中完全一致，匹配可靠
- **一次性操作**：合并完成后该命令不再需要，但保留作为工具可用
