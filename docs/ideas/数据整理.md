## 场景描述

本来想通过百度云盘的api上传文件到云盘，但是通过此方式只能上传到指定位置且生成分享链接还要收费，因此放弃通过api上传了。 使用人工上传文件，然后手动生成分享链接。

目前通过客户端批量生成分享链接导出的是csv格式，内格式如下
```csv
文件名,链接,提取码,分享时间,分享状态
小文艺口袋文库·知人系列,https://pan.baidu.com/s/1_UIWUi3l9_TxLoCyCV7wTA?pwd=6yk7,6yk7,2026-04-30 18:14,生成成功
```

在生成csv文件时，人工控制导出文名字是当前书籍的类型，例如 `传记.csv`。


在整理文件生成的json文件中，可知内容格式如下
```json
{
    "id": "727e3499-eeb0-435a-ae80-a285366373b7",
    "name": "小文艺口袋文库·知人系列",
    "author": "汉娜·阿伦特",
    "type": "传记",
    "pic": "传记/小文艺口袋文库·知人系列/d5825545.jpg",
    "sourceFolder": "小文艺口袋文库·知人系列（全7册）- 汉娜·阿伦特 等",
    "addedAt": "2026-04-30T09:28:58.160Z"
}
```

## 想要的效果

1. 通过type和name可以定位到csv文件中的链接 => url

2. 更新对应的json对象，新增字段bd_link: url

3. 因为是按类型来划分文件的，和json文件同级有多个csv文件

4. 在当前项目新建一个result文件夹(git忽略此文件夹)，然后生成的最终json文件在此保存一份副本

5. 副本json的名字为 "20260501-{条目数量}"
